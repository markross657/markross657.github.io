Project Idea

Overview and Motivation

The project I am going to outline over the next few paragraphs is one that I am currently already working on in my spare time.

The project is a budget management WebApp, which allows a user to enter all of their incomes and expenses, each with a Start Date, Frequency, Value and Title/Name.
From there the user will be able to view their budget on a Weekly, Fortnightly or Monthly scale, with the landing page providing a summary for the incoming and outgoing values of each Pay Cycle.

This project was inspired as I was previously using google calendars Schedule view to know which bills were coming out of my accounts each week, but found that this was an imperfect system and lacked a lot of functionality that I was looking for.
After hunting around to see if I could find an application that would let me do what i was looking for, I found a void in the marketplace as far as Web/Mobile apps go and decided to develop the application for my own use and portfolio.

Project Outline:

The budget app which I plan on developing will have several major functionalities, as outlined below:

Income / Expenses

The app will allow users to enter their incomes and expenses which will include a Name / Title, Value, Frequency and Starting Date.
The incomes and expenses will be able to be allocated against a bank account that the user enters, which will also have a Name, BSB and AccNo.

Simple Period Review

The application will allow users to see a general overview of their budget state on either a Weekly, Fortnightly or Monthly scale.
This will have a basic outline of how much money is coming in and out over that pay period at a quick glance.
From this page the user will also have a basic breakdown of what type of expenses are taking up which percentage of income mapped out into a pie chart for quick readability.

Detailed Period Review

The Detailed Period Review will show the user a breakdown of the exact income and expenses for that pay period.
The focus of this screen will be lower level details, allowing the user to know exactly where their income is from that week,
if they should have more than one source of income, and exactly which bills will be coming out of which bank accounts in that time period.
Again under this view the user will be able to see a breakdown entry by entry of the percentage breakdown allocated out to each entry.
For example, If a user has two incomes for that pay period, one is $400, one is $600, Income 1 will be listed as “Income1 - $400 - 40%”.

Calendar Budget Review

The calendar period review screen will display to the user in a similar fashion to the Google Calendar ‘Schedule’ page,
with the user being able to scroll down the list through the Weeks to see upcoming bills and payments that are due in the weeks to come.
This view will focus on fairly high level details, and will not display things like percentage breakdown.
The user will have the ability to drill down into a detailed view of any one week / pay period to see a detailed period review for that time period.

Tools and Technologies

It is in early stages of research for the tools and technologies although at this point it is looking like Angular
will be the framework used as it is versatile enough that I can utilise the same back end for both the Desktop and mobile platform,
and will just have to modify the front end to suit the platform.
As far as the hardware is concerned I will require the Desktop/Laptop to develop both the User Interfaces, Front and Back end.
With the mobile platform I will require several iOS devices with various screen resolutions (iPhone Max, iPhone Standard,
Differing generations of both). I will also need to test across various android platforms, although they are slightly more uniform in their screen resolutions.

Skills Requirements

The skill requirement will be for me to learn the Angular framework to a competent level,
as well as basic knowledge of building and designing User Interfaces.
From preliminary research it looks like the focus will be on using the AngularDart framework as i already
have a fairly decent level of understanding and competency with C#.
As most of the early stage testing on many platforms is to use a service like Lambatest.
Further down the road closer to taking the Application live, we would need to source actual devices to test on.

Outcomes

Should the project be successful not only would I of managed to provide myself with a product that i’ve been wanting for some time,
I would have also managed to add a valuable asset to my portfolio, increasing the likelihood of myself finding employment in a field which I have a lot of interest in.
